多链监控 - 使用手册

操作步骤、结果解释、API说明与案例演示

1. 操作步骤

  1. 输入待监控的地址列表，可选择TRON、ETH或BTC链。
  2. 点击"添加地址"，系统将为每个地址创建监控卡片。
  3. 点击"刷新状态"按钮，获取所有地址的最新链上数据。
  4. 查看各地址的余额、交易数、最后活跃时间等状态信息。
  5. 发现异常变化时，点击地址卡片进入详细分析。
  6. 选择"导出数据"将监控结果保存为JSON或CSV格式。

2. 结果解释

地址状态卡片

  • 地址：显示链类型图标和缩略地址
  • 余额：当前链上余额（TRX/ETH/BTC）
  • 交易数：历史交易总数
  • 最后活跃：最近一次链上活动时间
  • 状态标签：活跃/不活跃/风险

状态说明

  • 活跃（绿色）：最近24小时内有交易活动
  • 不活跃（黄色）：7天内无交易活动
  • 风险（红色）：检测到可疑行为或与风险地址交互

刷新说明

  • 手动刷新：点击刷新按钮主动获取最新数据
  • 刷新间隔：建议间隔至少1分钟，避免API限流
  • 批量刷新：一次性刷新所有已添加地址的状态

3. API密钥说明

本工具使用多链API，部分需要API密钥：

TRON链 - Tronscan API

无需API密钥，可直接使用。访问 Tronscan官网 了解详情。

以太坊链 - Etherscan API

需要API密钥，访问 Etherscan官网 注册获取。

比特币链 - Blockstream API

无需API密钥，可直接使用。API文档：https://blockstream.info/api

提示：ETH监控需要先配置Etherscan API密钥，TRON和BTC可直接使用。

4. 案例演示

案例背景

某诈骗案涉及多条链的收款地址，调查人员需要持续监控这些地址的资金变化。

操作过程

  1. 添加3个TRON地址、2个ETH地址、1个BTC地址到监控列表
  2. 首次刷新获取各地址状态，记录初始余额
  3. 每日定时刷新，对比余额变化
  4. 发现某TRON地址余额突然增加，点击进入详细分析
  5. 导出监控数据作为案件材料

结论

通过多链监控，成功追踪到诈骗资金的流入和转移，为案件侦破提供实时线索。
