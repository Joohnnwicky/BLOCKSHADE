地址聚类 - 使用手册

操作步骤、结果解释、API说明与案例演示

1. 操作步骤

  1. 输入多个待分析的区块链地址（支持TRON、ETH、BTC三种链）。
  2. 选择聚类方法：共输入聚类、时间关联聚类或综合聚类。
  3. 点击"开始聚类"，系统将通过多条链的API获取交易数据并分析关联性。
  4. 查看聚类结果，包括关联地址组、关联强度评分、关联依据。
  5. 选择导出格式（JSON/CSV/PDF），保存聚类分析报告。

2. 结果解释

聚类方法说明

  • 共输入聚类：识别向同一地址发送资金的多地址组（可能属于同一人）
  • 时间关联聚类：根据交易时间接近性识别关联地址
  • 综合聚类：结合多种方法进行综合分析，准确度更高

聚类结果

  • 聚类组：被判定为同一控制人的地址集合
  • 关联强度：高/中/低，表示聚类结论的可信度
  • 关联依据：具体的交易证据和时间戳
  • 链类型：标注每个地址所属的区块链

风险评估

  • 高风险聚类：多个地址与已知风险地址存在关联
  • 中风险聚类：聚类地址存在可疑行为模式
  • 低风险聚类：聚类地址行为正常

3. API密钥说明

本工具使用多链API，部分需要API密钥：

TRON链 - Tronscan API

无需API密钥，可直接使用。

以太坊链 - Etherscan API

需要API密钥，访问 Etherscan官网 注册获取。

比特币链 - Blockstream API

无需API密钥，可直接使用。

提示：为保证完整分析能力，建议配置Etherscan API密钥。TRON和BTC分析无需额外配置。

4. 案例演示

案例背景

某诈骗案发现多个收款地址分散在TRON和ETH链上，需要分析这些地址是否由同一团伙控制。

操作过程

  1. 输入5个TRON地址和3个ETH地址
  2. 选择综合聚类方法
  3. 分析发现其中3个TRON地址存在共输入模式
  4. 2个ETH地址在相似时间向同一交易所充币
  5. 生成聚类报告，确认8个地址属于2个控制组

结论

通过多链聚类分析，成功识别出两个诈骗团伙的地址集群，为案件侦破提供重要线索。
